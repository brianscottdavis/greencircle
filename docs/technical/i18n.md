# i18n Implementation Guide

## Supported locales
- en-AU (default)
- hi-IN (Hindi)
- zh-Hans (Simplified Chinese)
- fr-FR (French)

## App Router structure
Top-level locale segment:
- src/app/[locale]/(public)/...
- src/app/[locale]/(resident)/...
- src/app/[locale]/(crew)/...
- src/app/[locale]/(admin)/...

## Middleware behaviour
- Requests without locale segment redirect to best-match locale.
- Unsupported locale redirects to en-AU.
- Locale selection order:
  1) User.preferredLocale (if signed in)
  2) Accept-Language best match
  3) en-AU

## Translation keys
Use stable keys:
- Common.Save
- Dashboard.NextCollection
- Reports.RequestType.Overflow
- Status.Submitted

## UI consumption
- UI translates enums/status codes via keys (do not return translated labels unless needed).
- For dynamic content (Announcements/Challenges/Guides), query translations by locale with fallback.

## Database translation storage
- TranslationNamespace (e.g. Common, Dashboard, Reports, Notifications)
- Translation rows with (NamespaceId, Key, Locale, Value)

For admin-authored content:
- Store canonical record with TranslationKey for Title/Body
- Store per-locale values in Translation table using namespace Content.<entity> (e.g., Content.Announcement)
